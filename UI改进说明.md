# UI改进说明 - 会话删除按钮位置调整

## 改动内容

### 问题
原来的"清空所有会话"按钮在侧边栏底部,当会话列表较长时,按钮被遮挡,无法点击。

### 解决方案
将"清空所有会话"按钮从底部移到"会话历史"标题的右侧。

## 修改的文件

### 1. index.html
- 在"会话历史"标题旁边添加了垃圾桶按钮
- 删除了底部的操作按钮区域

### 2. style.css
- 添加了 `.section-header` 样式(标题和按钮的容器)
- 添加了 `.clear-all-btn-inline` 样式(内联垃圾桶按钮)
- 删除了 `.history-actions` 和 `.clear-all-btn` 旧样式
- 调整了 `.session-list-section` 布局

## 新按钮样式

- 位置: "会话历史"标题右侧
- 大小: 36x36px 圆角方形
- 颜色: 淡红色背景,红色图标
- 悬停效果: 变为红色背景,白色图标,轻微放大

## 测试步骤

1. 启动应用: `mvn spring-boot:run`
2. 访问: http://localhost:8080
3. 点击"历史记录"按钮打开侧边栏
4. 进行几次查询,创建会话历史
5. 查看"会话历史"标题右侧是否有垃圾桶按钮
6. 点击垃圾桶按钮,确认可以清空所有会话

## 效果
✅ 按钮始终可见,不会被遮挡
✅ 位置更合理,符合用户习惯
✅ 节省了底部空间,会话列表可以显示更多内容
